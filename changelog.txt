Version: 0.5.0
Date: 2026-07-17
  Features:
    - Added optional fire reactions for wood, coal, solid fuel, rocket fuel, filled fluid barrels, and flamethrower ammunition
    - Added item burning on transport belts and for stacks dropped on the ground
    - Added cargo reactions when supported items are inside destroyed chests, logistic containers, linked containers, or cargo wagons
    - Added separate runtime settings for each item type and location
  Changes:
    - Disabled the complete solid-item system by default to preserve existing-save behavior
    - Made coal burn more slowly and pollute more than wood
    - Made solid fuel burn without detonating, while rocket fuel and flamethrower ammunition can explode
    - Made filled barrels inherit the behavior of their contained fluid; empty barrels remain inert
    - Limited solid-item fire work per tick and avoided scanning burner fuel inventories
  Locale:
    - Added English and French names and descriptions for all solid-item settings
---------------------------------------------------------------------------------------------------
Version: 0.4.0
Date: 2026-07-17
  Features:
    - Added an optional ignition delay with a smoke warning
    - Added configurable ignition chances for each base damage type and a fallback for modded damage types
    - Added configurable minimum explosion energy, fallback fluid energy, pollution, fire intensity, and per-tick processing limit
    - Added fluid whitelist and blacklist settings
    - Added entity category settings for pipes, tanks, fluid wagons, production machines, pumps, and other fluid entities
    - Added persistent remote-interface controls for flammability, gas classification, and fluid energy
  Changes:
    - Combined energy from every flammable fluidbox in an entity
    - Added fallback energy for explicitly flammable fluids without a prototype fuel value
    - Calculated radial damage from the destroyed entity's footprint instead of only its center
    - Limited the number of queued explosions processed per tick to protect performance during very large chain reactions
    - Preserved remote fluid overrides across save and reload
  Locale:
    - Added English and French names and descriptions for all new settings
    - Corrected the French damage multiplier description
---------------------------------------------------------------------------------------------------
Version: 0.3.0
Date: 2026-07-17
  Changes:
    - Updated for Factorio 2.1
    - Migrated fluid access from the removed LuaEntity.fluidbox and LuaFluidBox APIs to the new LuaEntity fluid methods
    - Replaced the removed LuaEntity.neighbours API with LuaEntity.fluidbox_neighbours
    - Deferred chain-reaction waves by one tick to prevent C stack overflows in large pipe networks
    - Guarded zero-capacity and zero-radius calculations
  Locale:
    - Corrected the English damage multiplier description
